## 0. 目标与定位

在一小组高相关大盘股（默认七姐妹：AAPL, MSFT, GOOGL, AMZN, NVDA, META, TSLA）上做两件事：

- **A. 进出场规则研究**：建立有统计依据的入场 + 出场规则，诚实输出胜率、收益分布、失效概率、最坏情形。
- **B. 基本面因子验证**：用**假设驱动**的方式，验证少数基本面信号是否对**未来**收益有领先预测力，并把通过验证的因子作为 A 的进出场**条件**。

**两条定位红线（贯穿全文）：**
- 这不是"寻找最佳进出点"工具——最佳点只在事后存在。它是**规则校准器**：把进出从感觉变成带置信区间和样本数的经验分布。
- B **不是"相关性扫描器"**。绝不"列出所有基本面指标再算相关性挑显著的"——那是数据窥探（data dredging），是本工具明令禁止的反面做法（见第 1、4 节）。

**数据：** A 全程免费（yfinance 价格 + FRED 宏观）即可。B 需要 point-in-time 基本面数据，免费源做不干净，属于后期、需付费源（见第 5 节）。

## 1. 专属约束（违反则结论无效）

**个股方向（A）：**
1. **池化统计，不做单票统计（POOLED）。** 单票大事件样本太少（一只票够格大回撤通常仅 ~5–10 次）。规则胜率/收益分布必须把七只票事件**合并成一个样本池**统计；单票结果仅展示、不作结论，且标注该票事件数 N。
2. **必须折算"有效独立样本数 N_eff"。** 七姐妹高度相关（共同 AI/利率/科技 beta），不是七个独立赌注。须算相关矩阵并把名义 N 按相关性折算成 N_eff，所有置信区间基于 N_eff。**否则会把 ~2 倍证据误当 7 倍。**
3. **参数只能池化 + walk-forward 选取，禁止单票调参。** 任何入场/出场参数必须在合并样本上选、walk-forward 出样本验证；候选组数 >1 时对最优结果做 deflated Sharpe 折扣。**严禁**"在 TSLA 上调出夏普 3"式单票寻优。
4. **入场与出场成对回测。** 不允许只优化入场；每次评估的是"入场规则 × 出场规则"组合，出场对收益分布影响通常更大。

**基本面方向（B）：**
5. **假设先行，禁止全指标扫描。** 每个基本面因子必须先有一条**可证伪的经济逻辑假设**（写进代码注释/配置），再去验证。**禁止**"拉 N 个指标和股价算相关、挑显著的"这一动作。
6. **只认领先预测力，不认同期相关。** 必须测"t 时刻可观测的因子 → t+N 的远期收益"的关系（IC on forward returns），**同期相关性不作为任何证据**——同期相关大多是已被定价的信息，对你无用。
7. **强制 point-in-time，零前视。** 只能用"在该日期之前已公布"的基本面数据。财报滞后公布，须用真实公布日（report date / filing date），不是财报所属季度的期末日。
8. **多重检验必须计数并校正。** 记录一共验证了多少条假设，对显著性做 deflated Sharpe / FDR 校正；未校正的显著性在报告里必须标注"未校正、不可信"。

## 2. 模块清单（新增/扩展）

```
factors/
  signals.py          # A：入场信号（纯价格，可组合）
  fundamentals.py     # B：基本面因子（假设驱动，PIT，后期）
backtest/
  exits.py            # A：出场规则（止盈/止损/移动止损/时间止损/信号出场）
evaluation/
  rule_eval.py        # A：进出场规则池化评估（N_eff + bootstrap + 基准对比）
  factor_eval.py      # B：复用主文档 alphalens 评估，测领先 IC
```

### 2.1 factors/signals.py —— 入场信号（A，纯价格）
可组合布尔信号：`dip_from_high(pct)`、`ma_cross(fast, slow)`、`rsi_oversold(window, level)`、`vol_regime(...)`。支持 AND/OR 组合；组合数计入多重检验计数。

### 2.2 backtest/exits.py —— 出场规则（A，重点）
与入场成对传入回测：`take_profit(pct)`、`stop_loss(pct)`、`trailing_stop(pct)`、`time_stop(n_days)`、`signal_exit`。用 `vectorbt.Portfolio.from_signals`（`sl_stop/tp_stop`/自定义 exits）实现，含 fees + slippage。

### 2.3 evaluation/rule_eval.py —— 进出场规则评估（A，池化）
`evaluate_rule(entry, exit, tickers, ...) -> dict`：在七只票上各自触发"入场→出场"逐笔交易，合并成交易池统计（全部基于合并池 + N_eff + block bootstrap CI）：胜率、每笔收益分位分布、平均持有天数、**每笔途中最大浮亏(MAE)分布**、与"随机进场 + 持有相同平均天数"基准的超额、最差 5% 交易、最长连亏链。
强制措辞模板：
> "入场=[距高点回撤≥15%] × 出场=[移动止损20%]：合并 7 票共 N 笔（N_eff≈m），胜率 X%，每笔收益中位 +Y%、5 分位 −Z%，MAE 中位 −W%，相对随机基准超额约 +P 点（95% CI [a,b]）。结论：[弱/中等]优势，非择时圣杯。"
**禁止**输出"最佳入场点 $XXX"或目标价。

## 3. 基本面因子验证模块详规（B）

### 3.1 正确的工作流（反向，假设驱动）
**不是**"列出所有 → 算相关"，**而是**：
1. **写下一条假设**（含经济逻辑，可证伪）。
2. **做成 PIT 因子**：用"当时已公布"的数据，算成可逐日观测的时间序列。
3. **测领先预测力**：用 alphalens 算该因子对**未来** 1/5/21/63 日收益的 IC、分位收益（注意是未来，不是同期）。
4. **多重检验校正**：按验证过的假设总数做 deflated Sharpe / FDR。
5. **稳健性**：跨时间、跨这 7 票、扣成本后是否仍在。

### 3.2 候选假设清单（只做这几条，不扩展成指标扫描）

| 假设（经济逻辑） | 因子定义 | PIT 注意 | 如何测 |
|---|---|---|---|
| 盈利超预期会领先正漂移（PEAD，文献中真实存在） | SUE = (实际EPS − 预期EPS)/历史误差标准差 | 用财报**公布日**对齐，非季度末 | 公布后 1–63 日远期收益 IC/分位 |
| 盈利预期上修方向领先收益 | 分析师 EPS 预期的近 N 周变动方向 | 用预期快照的**时间戳** | 上修组 vs 下修组远期收益 |
| 估值过度拉伸后均值回归（自身历史分位） | 当前 PE/PS 在该票自身历史的百分位 | 滚动分位，不用未来数据 | 高分位 vs 低分位远期收益 |
| 盈利能力趋势（质量） | ROE / 毛利率的同比变化 | 用已公布财报 | 趋势改善 vs 恶化的远期收益 |
| **财报日历结构**（最实用、最干净） | 距下次财报天数 + 上次是否超预期 | **完全无前视**，日期是已知的 | 财报前后的波动率/收益结构，直接用于进出时点 |

> **重点提示给搭建者：** 对七姐妹这种重仓股，"距下次财报几天""上次是否超预期"这种**财报日历**信号，往往比一堆静态财务比率更直接影响短期进出场，而且天然无前视偏差——**B 阶段建议从财报日历这条开始**，它免费数据也相对可得、且最不容易做歪。

### 3.3 与 A 的衔接
通过验证的基本面因子，作为 A 进出场规则的**条件**（如"仅在估值低分位 + 距财报>10 天时启用回撤买入规则"），交给 `rule_eval.py` 做分条件的池化评估。

## 4. 执行计划（接在主文档 Phase 之后）

- **Phase S1**：`signals.py` + `exits.py`，单票（先 NVDA）跑通"入场→出场"逐笔交易，含成本。验收：画出每笔进出标记 + 收益分布。
- **Phase S2**：`rule_eval.py` 池化统计 + N_eff 折算 + bootstrap CI + 基准对比。验收：随机入场规则的"超额"≈0（sanity check）。
- **Phase S3**：walk-forward + deflated Sharpe 包裹规则选择。验收：每个"最优规则"都带出样本表现和折扣后夏普。
- **Phase F1（基本面，需付费 PIT 数据；先做最干净的一条）**：`fundamentals.py` 实现**财报日历**因子，测财报前后结构。验收：随机"假财报日"对照组 IC≈0。
- **Phase F2**：逐条加入 SUE / 预期上修 / 估值分位 / 质量趋势，每条单独测领先 IC，累计多重检验计数。验收：报告含 FDR/deflated 校正后的显著性。
- **Phase F3**：把通过验证的因子接成 A 的进出场条件，做分条件池化评估。

## 5. 硬性禁止清单（叠加在主文档之上）
1. **不要**单票寻优 → 池化 + walk-forward。
2. **不要**用名义 N 算显著性 → 用相关性折算的 N_eff。
3. **不要**只优化入场 → 入场×出场成对评估。
4. **不要**输出"最佳进出点 / 目标价" → 只给胜率与收益分布 + CI。
5. **不要**把七姐妹当独立样本 → 高度相关，证据量打折。
6. **不要**做全指标相关性扫描 → 每个基本面因子先有可证伪假设。
7. **不要**用同期相关当证据 → 只认对未来收益的领先预测力。
8. **不要**用财报所属季度期末日对齐 → 用真实公布日，杜绝前视偏差。
9. **不要**因聚焦少数票/少数因子就放松反过拟合 → 标的越少、因子越精挑，过拟合越危险。
10. **不要**把可视化做成可缩放/画线的行情终端 → 那是看盘工具，会把你拉回主观猜顶底，正是量化要替代的东西。

## 6. reports/ 可视化规格

### 6.1 定位铁律
本工具的图是**"决策与验证"图，不是"看盘"图**。K 线只作为进出场标记的**底图**出现，**不**做可缩放、画线、看形态的行情终端——那交给券商 App / TradingView。工具画图的唯一目的：让你**复核规则**（它到底在买卖什么）和**理解分布**（胜率、收益区间、最坏情形），而不是盯盘。

### 6.2 库选择（全免费）
- 静态图：`matplotlib`；K 线用 `mplfinance`；绩效/回撤 tear sheet 用 `quantstats`。
- 交互图（缩放、悬停看每笔交易）：`plotly`（candlestick + 标记）。
- 因子图：`alphalens-reloaded` 自带 tear sheet。
- 相关热力图：`matplotlib` / `seaborn`。

### 6.3 每张图规格

| 图 | 用途 | 库 | 所属 Phase | 强制标注 |
|---|---|---|---|---|
| 进出场标记图 (trade map) | 价格/K线底图上标入场▲、出场▼，每笔按盈亏着色 | mplfinance / plotly | S1 | 规则参数 + 笔数 |
| 单笔收益分布直方图 | 看胜率与分布形状 | matplotlib | S2 | N、N_eff、中位、5/95 分位 |
| 远期收益锥 / 分位扇形 | 概率区间，非单点预测 | matplotlib / plotly | S2 / F | horizon + 分位 + N_eff |
| MAE 分布图 | 途中最大浮亏分布，判断能否扛到出场 | matplotlib | S2 | N_eff、中位浮亏 |
| 回撤水下图 (underwater) | 净值回撤深度 + 持续时间 | quantstats | S3 | 最大回撤 + 恢复天数 |
| 规则 vs 基准对比 | 是否跑赢无脑基准（净值 + 分布并排） | quantstats / matplotlib | S2 / S3 | 必须并排基准 |

### 6.4 强制规范
- **任何涉及统计的图，标题或角标必须显示样本数 N（适用时附 N_eff）和置信区间**；没有样本数的统计图不许产出。
- **任何图都不得标注"目标价 / 最佳买卖点"**（与第 0、5 节一致）。
- 进出场标记图可用 K 线作底，但默认输出**静态 PNG** 到 `reports/{rule_name}/`；交互版（plotly html）为可选项。
- 每个评估函数运行后，**自动**把对应图存到 `reports/{rule_name}/`，并可选生成一个 `index.html` 汇总页。
- 配色与样式统一在 `reports/style.py`；深浅色都要可读。

## 8. 术语表（新手）
- **池化 (Pooling)**：把多只相似股票的交易事件合并成大样本统计，弥补单票事件太少。
- **有效独立样本数 (N_eff)**：考虑标的间相关性后"真正独立"的样本数；相关越高，N_eff 越小于名义 N。
- **入场×出场成对回测**：把"何时买"和"何时卖"作为整体规则测，两者共同决定收益分布。
- **途中最大浮亏 (MAE)**：一笔交易持有期内最深浮亏；决定你能否扛到出场。
- **移动止损 / 时间止损**：止损线随新高上移 / 持有满 N 天强制出场。
- **数据窥探 (Data Dredging / p-hacking)**：拉大量指标找"显著"关系；多重检验下大半是假阳性。本工具禁止。
- **Point-in-time (PIT)**：只用"该日期前已公布"的数据，杜绝前视偏差；财报须用公布日对齐。
- **领先 vs 同期 (Leading vs Contemporaneous)**：领先=t 时因子预示 t+N 收益（有用）；同期=两者同时动（多已被定价，无用）。
- **盈利超预期漂移 (PEAD)**：实际盈利超预期后，股价在公布后一段时间继续同向漂移的文献现象。
- **IC / 信息系数**：因子值与未来收益的截面相关；单因子 0.03–0.05 即可用。
- **FDR / Deflated Sharpe**：按检验/尝试次数扣除靠运气的显著性，治多重检验。
- **进出场标记图 (Trade Map)**：在价格/K线图上标出规则触发的每个买卖点，用于肉眼复核规则在交易什么。
- **水下图 (Underwater Plot)**：展示净值相对历史高点的回撤幅度随时间的变化。
- **远期收益锥 (Forward-Return Cone)**：把未来 N 期收益画成分位区间（如 10–90 分位），表达"概率区间"而非单点预测。
- **Tear Sheet**：一组标准化绩效或因子图表的合集（如 quantstats、alphalens 自动产出的报告页）。
